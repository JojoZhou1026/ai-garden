很多人现在一听到 AI Agent，第一反应还是：“你说的是小龙虾吗？”

这不奇怪。

虽然在我体感里，小龙虾已经像是上个朝代的事了，但它确实是很多人第一次听说“Agent”的入口。

所以这篇先不讲复杂技术，就把“小龙虾”这三个字说清楚。

一句话：

小龙虾，狭义上指的是 OpenClaw；广义上指的是后来一批“像 OpenClaw 一样，帮你调用 AI 去干活”的 Agent 壳子。

比喻一下：

如果大模型是发动机，小龙虾就是一辆乐高积木搭出来的小车。

它自己不产生智能，但它把发动机、方向盘、轮子、工具箱装到一起，让 AI 看起来真的能“跑起来”。

## 狭义的小龙虾：最早那只虾

最早大家说“小龙虾”，一般指的是 OpenClaw。

OpenClaw 是一个开源 AI Agent 项目。它的图标/形象和“龙虾”有关，所以中文圈很自然地把它叫成“小龙虾”。

它为什么会火？

核心原因不是它会聊天，而是它让大家第一次很直观地看到：

原来 AI 不只能在对话框里回答问题，还可以连接工具、访问文件、操作软件、执行一串任务。

以前我们对 AI 的想象，大多停留在：

> “帮我写一段话。”
>
> “帮我总结一份材料。”
>
> “帮我解释一个概念。”

但 OpenClaw 让很多人第一次看到另一种画面：

> “我给它一个目标，它自己去做。”

这就是它爆火的关键。

## 广义的小龙虾：后来一堆“虾”

后来“小龙虾”这个词就不只指 OpenClaw 了。

很多国内厂商、开发者、云服务商也开始跟进，做各种“虾”：

有的是帮你快速部署 OpenClaw。  
有的是把它接到国内模型和工具上。  
有的是做成更好用的界面。  
有的是干脆做自己的 Agent 产品，但借用了这波“小龙虾”的热度。  

所以广义上的“小龙虾”，更像一个民间叫法。

它不一定严格指某一个产品，而是指那类“把 AI 模型包起来，让它能连接工具、执行任务、看起来像个数字助理”的东西。

## 小龙虾的本质：它不是模型，是壳子

这里最容易误会的一点是：

小龙虾本身不是大模型。

它不是 Claude，也不是 GPT，也不是 DeepSeek。

它更像一个“壳子”或者“控制台”。

它背后还是要调用模型。模型负责理解你的话、判断下一步怎么做、生成内容；小龙虾负责把这些能力接到工具、文件、网页、系统操作上。

可以这样理解：

| 部分 | 像什么 | 负责什么 |
|---|---|---|
| 大模型 | 大脑 | 理解、推理、生成 |
| 小龙虾这类 Agent 壳子 | 身体和工具箱 | 连接工具、执行步骤、调用系统 |
| 用户 | 目标设定者 | 告诉它要完成什么、边界在哪里 |

所以小龙虾厉害的地方，不是“它有一个神秘新大脑”。

它厉害的地方在于：它把大模型接进了真实工具世界。

这也是为什么大家会觉得它突然“像个人在干活”。

## 为什么小龙虾重要？

小龙虾肯定不是最成熟的 Agent 产品。

它也有很多问题：部署麻烦、安全风险高、容易误操作、普通人不一定用得明白。

但它的重要性在于：

它是 Agent 这个模式出圈的分界点之一。

在它之前，很多人听到 Agent，脑子里还是论文、框架、开发者工具、技术概念。

在它之后，很多人终于有了一个具体印象：

哦，Agent 大概就是那种“你交代一件事，它能自己去调用工具推进”的东西。

## 那现在还需要折腾小龙虾吗？

没必要。

可以把小龙虾想象成一辆用乐高积木搭出来的车。

它很好玩。对初学者来说，亲手搭一遍，确实能更直观地理解背后的逻辑：模型怎么被调用，工具怎么被连接，任务怎么一步步推进。

但问题是，现在市场上已经有成品车了。

如果你的目标是学习原理，搭乐高很有价值。

如果你的目标是直接干活，没必要从搭车开始。直接开车就行。

现在已经有更成熟的 Agent 产品，比如 Claude Code、Codex、Manus，以及国内的 WorkBuddy、Trae、Qoder CN、通义灵码。

当然，车也分三六九等。

我个人的看法是：要用就尽量用最好的。但每个人的任务不同、预算不同、电脑环境不同，也可以多试几个，最后选一个真正适合自己的。

## 一句话带走

狭义的小龙虾，是 OpenClaw。

广义的小龙虾，是一类把大模型包起来、让 AI 能连接工具和执行任务的 Agent 壳子。

但现在真要干活，不必再回头研究怎么养虾。直接下载市面上的热门 Agent 工具用起来，先挑一个适合自己的。

更重要的是，不要把能力绑死在某一个工具上。

因为真正决定你能不能把 Agent 用起来的，不是你今天用的是小龙虾、Claude Code、Codex，还是别的什么新工具，而是你有没有形成一套好的协作习惯：怎么交代目标，怎么补充背景，怎么拆任务，怎么检查结果，怎么在 AI 跑偏时把它拉回来。

工具会一代一代换，但这些习惯会跟着你走。

这一点可以接着看《看真事儿》里的这篇：[账号被封之后，我才发现自己真正依赖的不是 Claude Code](https://jojomind.com/posts/06-%E8%B4%A6%E5%8F%B7%E8%A2%AB%E5%B0%81%E4%B9%8B%E5%90%8E%E6%88%91%E6%89%8D%E5%8F%91%E7%8E%B0%E8%87%AA%E5%B7%B1%E7%9C%9F%E6%AD%A3%E4%BE%9D%E8%B5%96%E7%9A%84%E4%B8%8D%E6%98%AFClaude-Code/)。

那篇讲的就是：最重要的不是押中哪个 AI 工具，而是培养一种能迁移到任何工具上的 AI 协作能力。
